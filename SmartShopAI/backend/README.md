# SmartShopAI Backend

FastAPI + SQLite backend for SmartShopAI. The original AI guide was a local rule-search flow. This version upgrades the guide to a GraphRAG architecture while keeping the original rule retrieval as a hard fallback.

## What Changed

- SQLite remains the source of truth for product cards, product IDs, prices, stock, SKU, images, cart, orders, FAQ, and reviews.
- Milvus handles dense semantic retrieval over product chunks.
- Neo4j stores the product knowledge graph for category, brand, SKU, feature, scenario, review, FAQ, and similarity relationships.
- Poe/Qwen `qwen3.6-plus` generates natural language shopping guidance from retrieved context.
- The original keyword and rule retrieval still runs when LLM, Milvus, embedding, or Neo4j is unavailable.
- Post-generation verification checks hallucination risks before returning text and product cards.

## Setup

```powershell
cd J:\full-stack\SmartShopAI\backend
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
Copy-Item .env.example .env
```

Edit `.env`:

```env
POE_API_KEY=your_real_poe_key
POE_BASE_URL=your_openai_compatible_base_url
QWEN_MODEL=qwen3.6-plus

MILVUS_URI=http://127.0.0.1:19530
MILVUS_COLLECTION=smartshop_products

NEO4J_URI=bolt://127.0.0.1:7687
NEO4J_USER=neo4j
NEO4J_PASSWORD=your_password
```

If these services are not configured, the backend still starts and the AI guide falls back to local rules.

## Data Build

Import product JSON into SQLite:

```powershell
python scripts\import_dataset.py --dataset ..\app\ecommerce_agent_dataset --db .\data\smartshop.db --clean .\data\products_clean.json
```

Build text RAG chunks:

```powershell
python scripts\build_rag_index.py --db .\data\smartshop.db
```

Build Milvus vectors:

```powershell
python scripts\build_milvus_index.py --db .\data\smartshop.db --reset
```

Build Neo4j graph:

```powershell
python scripts\build_neo4j_graph.py --db .\data\smartshop.db --reset
```

Milvus and Neo4j scripts fail with clear messages if the service or dependencies are unavailable; this does not affect basic backend startup.

## AI Guide Flow

```text
user query / image hint
-> IntelligentQueryRouter
-> HybridRetrieval or GraphRAGRetrieval or combined graph_hybrid
-> prompt construction for qwen3.6-plus
-> PostGenerationVerifier
-> SSE assistant_text_delta + products + strategy + fallback_used + verification_result
```

The frontend keeps the current SSE chat UI and now supports two modes:

- `AI导购`: natural language recommendations, explanation, and comparison.
- `用户检索`: structured product search with more result cards.

## Hallucination Controls

LLM hallucination risks include fabricated products, prices, stock, SKU, unsupported features, budget violations, unsupported graph relationships, and ranking conflicts.

Controls:

- Prompt constraints require the model to use only retrieved context and candidate products.
- Product card fields are locked to SQLite, never generated by the LLM.
- The verifier removes products outside candidate Top 10.
- The verifier checks budget and category constraints.
- Text prices, stock, and SKU claims are corrected to database-backed wording.
- Unsupported feature claims are weakened to safe wording.
- If verification removes every product, the agent falls back to local rule retrieval.
